## üîÑ Complete Workflow (As Implemented)

### Upload & Start
1. User uploads file ‚Üí `POST /api/upload`
2. Backend acknowledges upload (does NOT auto-start)
3. User clicks "Start Conversion" ‚Üí `POST /api/start-conversion`
4. Conversation Agent ‚Üí Conversion Agent: `detect_format`
5. Conversation Agent validates metadata (requests if missing)

### Conversion
6. Conversation Agent ‚Üí Conversion Agent: `run_conversion`
7. Conversion Agent converts to NWB
8. Conversation Agent ‚Üí Evaluation Agent: `run_validation`

### Validation Outcomes

#### PASSED (No Issues)
- Evaluation Agent generates PDF report
- Conversation Agent sets `status=COMPLETED`
- User downloads NWB + PDF
- **END**

#### PASSED_WITH_ISSUES (Warnings Only)
- Evaluation Agent generates PDF with warnings highlighted
- Conversation Agent sets `status=AWAITING_USER_INPUT`
- Frontend shows: "üîß Improve File" vs "‚úÖ Accept As-Is"
- User chooses:
  - **ACCEPT**: Finalize ‚Üí Download NWB + PDF ‚Üí **END**
  - **IMPROVE**:
    - Evaluation Agent categorizes issues
    - If user input needed ‚Üí Request data
    - Conversation Agent ‚Üí Conversion Agent: `apply_corrections`
    - Loop back to step 8 (validation)

#### FAILED (Critical/Error Issues)
- Evaluation Agent generates JSON correction context
- Conversation Agent analyzes with LLM
- Frontend shows: "üîÑ Retry with Corrections" vs "‚ùå Decline"
- User chooses:
  - **DECLINE**: Finalize ‚Üí Download NWB + JSON ‚Üí **END**
  - **APPROVE**:
    - Categorize issues (auto-fixable vs user input)
    - Request user input if needed
    - Apply corrections and reconvert
    - Loop back to step 8 (validation)
